Searx
b"Searx (/s\xc9\x9c\xcb\x90rks/; stylized as searX) is a free and open-source metasearch engine,[4] available under the GNU Affero General Public License version 3, with the aim of protecting the privacy of its users.[5][6][7] To this end, Searx does not share users' IP addresses or search history with the search engines from which it gathers results. Tracking cookies served by the search engines are blocked, preventing user-profiling-based results modification.[8][9] By default, Searx queries are submitted via HTTP POST,[a] to prevent users' query keywords from appearing in webserver logs.[10][11] Searx was inspired by the Seeks project,[10] though it does not implement Seeks' peer-to-peer user-sourced results ranking.\n"

\n**Searx** ([/s\xc9\x9c\xcb\x90rks/](/wiki/Help:IPA/English "Help:IPA/English"); stylized as\n**searX** ) is a [free](/wiki/Free_software "Free software") and [open-\nsource](/wiki/Open-source "Open-source") [metasearch\nengine](/wiki/Metasearch_engine "Metasearch engine"),[4] available under the\n[GNU Affero General Public License version\n3](/wiki/GNU_Affero_General_Public_License "GNU Affero General Public\nLicense"), with the aim of protecting the [privacy](/wiki/Privacy "Privacy")\nof its users.[5][6][7] To this end, Searx does not share users\' [IP\naddresses](/wiki/IP_address "IP address") or search history with the [search\nengines](/wiki/Search_engine "Search engine") from which it gathers results.\n[Tracking cookies](/wiki/HTTP_cookie#Tracking "HTTP cookie") served by the\nsearch engines are blocked, preventing [user-profiling-based results\nmodification](/wiki/Filter_bubble "Filter bubble").[8][9] By default, Searx\nqueries are submitted via [HTTP POST](/wiki/HTTP_POST "HTTP POST"),[a] to\nprevent users\' query keywords from appearing in webserver logs.[10][11] Searx\nwas inspired by the [Seeks](/wiki/Seeks "Seeks") project,[10] though it does\nnot implement Seeks\' [peer-to-peer](/wiki/Peer-to-peer "Peer-to-peer") user-\nsourced results [ranking](/wiki/Ranking_\\(information_retrieval\\) "Ranking\n\\(information retrieval\\)").\n\nEach search result is given as a direct link to the respective site, rather\nthan a [tracked redirect link](/wiki/Internet_tracking "Internet tracking") as\nused by [Google](/wiki/Google_Search "Google Search"). In addition, when\navailable, these direct links are accompanied by cached and/or proxied links\nthat allow viewing results pages without actually visiting the sites in\nquestion. The cached links point to saved versions of a page on the [Wayback\nMachine](/wiki/Wayback_Machine "Wayback Machine"), while the proxied links\nallow viewing the current live page via a Searx-based [web\nproxy](/wiki/Web_Proxy "Web Proxy"). In addition to the general search, the\nengine also features tabs to search within specific domains: files, images,\n[IT](/wiki/Information_technology "Information technology"), maps, music,\nnews, science, social media, and videos.[12][13]\n\nUsers can run private instances of Searx on their own\n[computer](/wiki/Localhost "Localhost"), but there are also many public, user-\nrun, Searx instances,[14] some of which are available as\n[Tor](/wiki/Tor_\\(anonymity_network\\) "Tor \\(anonymity network\\)") [hidden\nservices](/wiki/Hidden_service "Hidden service").[14] Meta-Searx instances can\nalso be used to forward the search query to a random public instance.[14] A\npublic [API](/wiki/API "API") is available for Searx,[15][16] as well as\n[Firefox](/wiki/Firefox "Firefox") search provider plugins.[17]\n\n## Search engines and other\nsettings[[edit](/w/index.php?title=Searx&action=edit&section=1 "Edit section:\nSearch engines and other settings")]\n\nAcross all categories, Searx can fetch search results from about 82 different\nengines. This includes major search engines and site-specific searches like\n[Bing](/wiki/Bing_\\(search_engine\\) "Bing \\(search engine\\)"),\n[Google](/wiki/Google_Search "Google Search"), [Reddit](/wiki/Reddit\n"Reddit"), [Wikipedia](/wiki/Wikipedia "Wikipedia"),\n[Yahoo](/wiki/Yahoo!_Search "Yahoo! Search"), and [Yandex](/wiki/Yandex_Search\n"Yandex Search").[18] The engines used for each search category can be set via\na "preferences" interface, and these settings will be saved in a cookie in the\nuser\'s web browser, rather than on the server side, since for privacy reasons,\nSearx does not implement a [user login](/wiki/Login "Login") model. Other\nsettings such as the search interface language and the search results language\n(over 20 languages are available) can be set the same way.[11]\n\nIn addition to the preferences cookie, it is possible on each query to modify\nthe engines used, search categories selected, and/or languages to search in by\nspecifying one or more of the following textual search operators _before_ the\nsearch keywords.[19]\n\n  * `! _category_` \xe2\x80\x94 Search the specified _category_ instead of the default ones.\n  * `? _category_` \xe2\x80\x94 Search the specified _category_ in addition to the default ones.\n  * `! _engine_` \xe2\x80\x94 Search the specified _engine_ instead of the default ones.\n  * `? _engine_` \xe2\x80\x94 Search the specified _engine_ in addition to the default ones.\n  * `: _language_` \xe2\x80\x94 Search for results in the specified _language_ instead of the default one.\n\nThe `!` and `?` operators can be specified more than once to select multiple\ncategories or engines, for example `!google !deviantart ?images :japanese\ncow`.\n\n## Instances[[edit](/w/index.php?title=Searx&action=edit&section=2 "Edit\nsection: Instances")]\n\nAny user may run their own instance of Searx,[20] which can be done to\nmaximize privacy, avoid congestion on public instances, preserve customized\nsettings (even if browser cookies are cleared), allow auditing of the source\ncode being run, and more.[21] Users may include their Searx instances on the\neditable list of all public instances, or keep them private.[18][21] It is\nalso possible to add custom search engines to a [self-hosted](/wiki/Self-\nhosting_\\(web_services\\) "Self-hosting \\(web services\\)") instance that are\nnot available on the public instances.[22]\n\nAnother reason to use different Searx instances, or to run one\'s own, is that,\nas of 2019, Google has begun to block some instances, including some of the IP\naddresses used by searx.me (former instance run by the developer), from\nquerying it, resulting in a "google (unexpected crash: CAPTCHA required)"\nerror.[23] In response, some instances have been modified to silently skip\ntrying to search with Google, even when it\'s the only engine\nspecified.[24][25]\n\n  \n\n## See also[[edit](/w/index.php?title=Searx&action=edit&section=3 "Edit\nsection: See also")]\n\n  *
